-   STUDIO NEXS

# 4. オフライン対応方針

イベント会場は地下会場であり、500人規模で通信障害が起きる可能性が高い。

そのためオフライン対応を前提とする。

構成：

スマホ ↓ IndexedDB ↓ Firestore Offline ↓ Firebase同期

注文登録時：

1.  スタッフ入力

2.  ローカル保存

3.  通信復帰時自動同期

編集ではなく履歴追加方式を採用する。

禁止：

注文上書き

採用：

注文追加 注文取消

# 7. スタッフ売上管理

スタッフによるボトル販売を記録する。

注文入力者と売上担当者を分離する。

保持情報：

-   入力者
-   売上担当者
-   提供担当
-   会計担当

例：

VIP-B

シャンパンA

50,000円
